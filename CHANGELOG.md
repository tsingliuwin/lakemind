## [0.5.3] - 2026-07-11

### 新增

#### 图表可视化增强（多序列与单位展示）
- **双 Y 轴支持**：当多条序列不在同一数量级时（如“金额”百万级 vs“转化率”百分比），单轴会把小量级序列压成一条直线。LLM 可通过新增的 `right_y_fields` 参数将不同量级的序列分到右轴，由 LLM 根据分析场景主动选择单轴或双轴。
- **图表单位展示**：新增 `y_field_labels` 参数，让图例与轴名直接显示带单位的可读名（如“销售额(万元)”）。双轴时读者一眼分清左右轴各是什么量级，单序列图表也会在轴名上带单位。
- **可视化经验沉淀**：将“多序列同量级用单轴、量级/单位根本不同才分双轴”的判断经验，以及“图表要显示单位”的呈现纪律写入数据呈现准则。

## [0.5.2] - 2026-07-11

### 修复

#### 用户界面与全局主题优化 (浅色模式适配)
- **全局浅色主题重构**：重构并引入了 Slate & Indigo 全局浅色配色系统，彻底解决侧边栏选中节点文字/时间戳对比度过低看不清的问题。
- **对话与代码对比度提升**：修复了行内代码文本太浅、CodeMirror SQL 编辑器在浅色模式下误载入深色语法主题而导致关键字/字符串显示苍白难读的问题。
- **上下文容量面板自适应**：支持在深/浅色模式间自适应展现卡片阴影与用量圆点色，用量填充条全面切换为主题态变量。
- **复制 SQL 交互简化**：将表结构属性页 DDL 定义下方的“复制 SQL”文字边框按钮，改造为右上角绝对定位悬浮的扁平无边框 SVG 图标按钮，成功后展现绿勾反馈。
- **选项按钮图标对齐**：使用 Flexbox 精确对齐输入框下方下拉选项按钮（模型/优先/确认模式）中图标与文字的几何中线，解决错位问题。
- **全屏图表视觉去噪**：去除全屏图表容器的浮动 card 冗余外边框线与白底，防止与 header 边框形成错乱的“双横线”效果；全屏 overlay 亦完美适配浅色毛玻璃模糊背景。

## [0.5.0] - 2026-07-10

### 新增

#### 数据分析 Agent
- 引入全局分析准则库（OKF bundle）：将"分析准则"从扁平糊在系统提示词里的静态文本，重构为遵循 [OKF v0.1 规范](https://github.com/GoogleCloudPlatform/knowledge-catalog/blob/main/okf/SPEC.md) 的分层、标签化、可检索、按需注入的活知识库。准则随软件分发于 `~/.lakemind/tenets/`，所有工作区共享，是产品资产而非每工作区从零积累。
- 分层架构：PREAMBLE 只保留跨行业普适的核心准则（数据纪律红线）；行业案例（教育、旅游、房地产…）与分析主题准则（转化、增长…）按 frontmatter 标签多维分类（`industry:` / `topic:` / `error-type:` / `severity:`），按需检索。
- 新增 2 个 Agent 工具（共 16 个）：`search_tenets`（关键词检索准则标题/描述/标签/正文）、`load_tenets`（支持渐进式披露——不带参数返回目录大纲，带 concept_id 精读单条，带标签批量加载）。
- 首次启动自动初始化种子准则库（已存在则跳过，尊重用户编辑）。
- 扩展性：新增行业/主题只需添加一个带 frontmatter 的 Markdown 文件并更新对应 index.md，零代码。bundle 可 git 化、可 zip 分发。

### 变更

#### 数据分析 Agent
- 重构系统提示词（PREAMBLE）的数据纪律章节：回退此前硬编码在其中的教育行业具体案例（面向全球用户不应常驻单一行业案例），改为跨行业普适的抽象表述；同时引导 Agent 在涉及特定行业或分析主题时主动检索准则库。该教育案例（"3,546 单 99 元体验课"归因错误）完整保留于 `industry/education.md`，按标签被检索。

- 大模型配置卡片最大宽度扩展至 `900px`，对话框容器支持水平缩放并限制默认最大宽度。
- 清理了多处歧义图标：如将对话与 SQL 控制栏的关闭交叉图标 `X` 替换为垃圾桶图标，避免与面板关闭混淆。

## [0.3.1] - 2026-07-10

### 新增

#### 应用更新
- 接入应用内真自动更新：在 Tauri 中集成 `tauri-plugin-updater` 与 `tauri-plugin-process`，点击"检查更新"后可直接在应用内完成下载（带实时进度条）→ 签名校验 → 一键安装并重启，无需再手动去网页下载安装包。安装包经 minisign 私钥签名，更新 manifest 托管于官网并随发版自动生成。检测失败时自动回退到打开下载页。

#### 模型配置
- 模型供应商配置新增"测试连接"按钮：针对当前表单值（含未保存的改动）发送最小化请求，即时验证 endpoint / API Key / API 格式 / 模型 ID 是否可用，避免到正式对话时才发现配置错误。测试结果以绿/红圆点显示在供应商列表，失败时给出按错误类型分类的中文提示（鉴权失败、模型未找到、额度耗尽、服务端故障等）并支持复制原始错误日志。

### 修复

#### 数据导入
- 修复 GBK/GB18030 编码 CSV 导入后表头变成 `column00…columnNN` 且 0 行数据的问题（中文 Windows Excel/WPS 导出的 CSV 默认即此编码）。导入前新增字节级 UTF-8 探测：检测到非 UTF-8 时优先以 `encoding='zh_CN.gbk'` 读取，避免被 `ignore_errors` 策略"吞错"生成空表。同时收紧 CSV 候选校验，拒绝表头全为 DuckDB 自动占位列名的候选，作为兜底安全网。

#### 外部数据库
- 修复零拷贝 VIEW 方式注册的外部表（postgres/mysql）在连接账号无 SELECT 权限时"注册成功却查看数据报错"的问题。注册阶段新增 `SELECT ... LIMIT 1` 真实读取探针（强制触发表级权限校验），权限不足时立即返回清晰的中文提示并附带 GRANT SELECT 建议，而非让错误推迟到首次查看数据。

## [0.2.0] - 2026-07-06

本版本聚焦于外部数据库集成、数据可视化、远程表物化、动态知识系统（OKF）与运行稳定性，是一个功能与可靠性全面升级的大版本。

### 新增

#### 外部数据库
- 支持连接外部 MySQL、PostgreSQL 数据库，并提供虚拟文件夹式数据浏览器。
- 新增 SQLite 作为外部数据库连接类型，启用 WAL 模式以支持并发读取。
- 数据库连接设置支持 Connection URI 智能解析与一键快速填充。
- 数据库 schema 缓存与连接生命周期管理，提升浏览与查询响应速度。
- 工作区切换后自动重连，并在左侧栏以独立图标与字母徽标区分连接类型。

#### 数据可视化
- 对话中可直接生成 ECharts 图表，支持柱状图、折线图、饼图、散点图，并可在图表 tab 间切换类型。
- 新增漏斗图（转化漏斗分析）与仪表盘（单值指标展示）。
- 图表配色与坐标轴/tooltip/图例全面适配深色主题，图例与轴不再重叠。

#### 远程表物化与查询
- 新增 `materialize_remote_table` 工具，可远程表本地物化并支持实时进度（基于 DuckDB Appender）。
- 物化支持分区物化、增量更新、超时控制与断点续传。
- 支持本地/远程表名模糊匹配，并在工具输出中注入数据库类型（postgres/mysql）以指导 Agent 选择正确的查询通道。
- 引入双路查询策略（采样 vs 全量）：本地采样表默认用于探索，正式分析时提示全量物化路径，避免基于采样的误导性聚合结果。
- 为聚合误导场景增加查询拦截与全量推送建议。

#### 智能体与运行时
- 429 限流时自动指数退避重试，区分瞬时限流与配额/余额耗尽（后者不再重试）。
- 查询执行引入软+硬超时限制与锁竞争优化，避免长时间阻塞 UI。
- 对话流式输出支持随时停止（发送按钮变为红色停止按钮）。
- Token 用量统计：累计与单轮用量、上下文容量面板（进度条 + 占比 + 加权平均缓存命中率），数据持久化到 SQLite。

#### 知识系统（OKF）
- 实现 OKF 动态知识系统：基于 Markdown + YAML 的多目录知识沉淀（sources / pipelines / tables / views / concepts），Agent 主动注入记忆。
- 支持写入时自动创建缺失的 OKF 文件与目录，打通 concepts 全局业务概念的主动沉淀。
- OKF 写入自动 git 追踪、markdown 自动修复去重，并提供动态标题检测与记忆摘要。
- 汉化 OKF 相关工具标签、参数与结果抽屉展示，统一工具图标。

#### 界面与体验
- 工作区记忆：重启后自动重开上次使用的工作区。
- 数据库设置空状态可深链跳转，文件导入支持文件/文件夹双按钮并提示空扫描。
- 左侧栏支持折叠按钮，配置页新增空状态图标。
- 数据库表/视图在侧边栏改用图标徽标，支持按行折叠子树。
- SQL 定义在右侧面板可折叠查看。

#### 文档与打包
- 新增 VitePress 文档站点，阐述产品理念与技术栈。
- 升级 DuckDB 至 1.5.4 并加入 DuckLake 扩展。

### 修复

- 修复 429 配额/余额耗尽被误判为瞬时限流而反复重试的问题。
- 修复 429 重试耗尽后静默结束而不报错的问题。
- 修复物化分区时 DuckDB 时间分区边界类型错误与自适应 bucket 宽度问题。
- 修复工作区切换时的竞态条件与表目录污染。
- 修复缓存命中率超过 100% 的问题，平均命中率改为真实加权平均。
- 修复上下文容量进度条随新消息缩水的问题（改用历史峰值）。
- 修复对话数据面板 Token 用量未跨轮累积的问题。
- 修复未分析数据库表显示无意义行数（-1）的问题，并在 Postgres 行统计不可用时回退 COUNT(*)。
- 修复 Postgres/MySQL 的 `LIMIT` pushdown 被子查询包装破坏的问题。
- 修复文件选择器中文路径丢失的问题。
- 修复侧边栏收起时标题栏残留历史箭头。
- 修复工作区折叠/展开未包含数据库分区。
- 修复 MySQL 切换时的 SSL Mode 布局偏移。
- 修复 WebKit 下 CSS 变量渲染异常导致的状态图标不可见。
- 修复对话框工具栏溢出与按钮被裁切。
- 修复 OKF 类别目录自动回退与 sources 源文件解析、工作空间目录解析（写入正确绝对路径）。
- 修复启动时短暂白屏：在 index.html 内联首屏深色背景与启动占位，消除 JS 加载完成前的白闪。
