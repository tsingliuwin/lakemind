# LakeMind 分析准则库

本准则是 LakeMind 从真实案例中提炼的数据分析方法论体系。它随软件分发，所有工作区共享。动手分析前，先查这里有没有相关准则，比现想更可靠。

浏览方式：先看本大纲，定位到相关准则后，用 `load_tenets` 按 concept ID 加载全文精读；不确定关键词时用 `search_tenets` 检索。

## 核心准则

核心准则分为**总则**（根本原则）、**数据纪律红线**、**数据安全准则**、**分则**（按阶段）和**准则变更准则**（元规则）：

### 总则
* [总则：根本原则](core/general-principles.md) — 实事求是、数据为证、归属正确、最小假设、先理解后分析、分析可复现。

### 数据纪律红线
* [数据纪律](core/data-discipline.md) — 不可触碰的五条红线：禁止编造、禁止张冠李戴、三问自检、禁止截断、先怀疑自己。

### 数据安全准则
* [数据安全与隐私](core/data-security.md) — PII 识别、数据脱敏、数据分级、最小化原则、未成年人数据特化。纪律管"数字真实性"，安全管"数据使用合规"。

### 分则（按数据分析阶段）
* [数据画像](core/data-profiling.md) — 如何了解你的数据：时间范围、数据质量、字段含义、跨表一致性、字段类型。
* [指标体系与口径](core/data-metrics.md) — 动手分析前先把指标定义清楚：五要素、口径文档化、同名不同义陷阱。
* [数据清洗](core/data-cleaning.md) — 从原始数据到高质量数据表：脏数据识别、清洗策略、JOIN 爆炸、可追溯原则。
* [数据分析](core/data-analysis.md) — 从数据到洞察：假设驱动、多维验证、统计陷阱规避、时间序列陷阱。
* [数据呈现](core/data-presentation.md) — 如何展示分析结果：图表选择、数字精度、结论表达。

### 准则变更准则
* [准则的准则](core/meta-governance.md) — 准则本身的修改也需要规范：生命周期、来源要求、层级关系。
