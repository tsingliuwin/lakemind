# 数据安全与隐私准则

> 与数据纪律红线（`core/data-discipline.md`）并列。**纪律管"数字真实性"，安全管"数据使用合规"**——两者都是横切所有阶段的底线，不可触碰。

真实业务数据里常常包含个人信息（PII）：手机号、身份证、姓名、地址、银行卡。分析这些数据时，如果你把含 PII 的结果集明文贴进对话、或 `SELECT *` 拉了全表，就可能造成隐私泄露。本准则定义数据分析中的安全底线。

## 1. PII 识别——先认得出，才保护得了

拿到数据后，优先识别哪些字段是个人信息（PII，Personally Identifiable Information）。常见 PII 及识别特征：

| PII 类型 | 识别特征（字段名暗示 / 值特征） |
|---|---|
| 手机号 | 字段名含 `phone`/`mobile`/`tel`；11 位数字；正则 `^1[3-9]\d{9}$` |
| 身份证 | 字段名含 `id_card`/`idno`/`cert`；18 位（末位可能是 X） |
| 姓名 | 字段名含 `name`/`real_name`；中文 2-4 字 |
| 邮箱 | 字段名含 `email`/`mail`；含 `@` |
| 地址 | 字段名含 `address`/`addr`；长文本含省市街道 |
| 银行卡 | 字段名含 `card_no`/`bank_card`；16-19 位数字 |
| 设备号 | 字段名含 `device`/`imei`/`idfa`/`uuid` |

**怎么做**：数据画像阶段（见 `core/data-profiling.md`）就用 `describe_table` + `sample_data` 扫一遍字段名和样例值，标记出 PII 字段。识别到 PII 后，后续操作按本准则处理。

## 2. 数据脱敏——查询 PII 时默认脱敏

需要用到 PII 字段做分析时（如按手机号去重统计用户数），**默认脱敏后再输出**，而不是把明文贴进结论。

| 场景 | 策略 | 示例 |
|---|---|---|
| 只需用 PII 做去重/关联，不需看明文 | 用 `HASH()` 或只取不可逆摘要 | `COUNT(DISTINCT hash(phone))` |
| 需要展示但不需要全量 | 中间脱敏，只露首尾 | `SUBSTR(phone,1,3) \|\| '****' \|\| SUBSTR(phone,8,4)` → `138****1234` |
| 用户明确要求查看完整 PII | 确认意图后提供，并在结论中提示"含敏感信息" | 仅当用户明确说"我要看完整手机号"时 |

**关键**：脱敏是默认行为，明文是例外。例外需要用户明确要求。

## 3. 数据分级——不同级别，不同态度

| 级别 | 含义 | 处理态度 |
|---|---|---|
| 公开 | 已对外公开的数据（如官网课程价格） | 无特殊要求 |
| 内部 | 公司内部业务数据（如订单量、销售额） | 正常分析，但结论不含 PII |
| 敏感 | 含个人信息的业务数据（如用户表、订单含手机号） | 必须脱敏，`SELECT *` 谨慎 |
| 高敏 | 含强 PII 的数据（身份证、银行卡、未成年人信息） | 最高警惕，最小化查询，严禁明文导出 |

**怎么做**：拿到数据先判断级别。级别越高，后续每一步（查询、清洗、呈现、导出）都要越谨慎。

## 4. 最小化原则——只查需要的，不多拉一个字段

- **不 `SELECT *` 含 PII 的表**。只 SELECT 分析真正需要的列。如果你只要统计订单数，不要把用户手机号一起拉出来。
- **导出文件前脱敏**。如果要把分析结果导出成文件（如 CSV），导出前确认结果集不含 PII，或已脱敏。
- **样例数据也要警惕**。`sample_data` 返回的行里可能含 PII，不要把含手机号/身份证的样例原样贴进对话。

## 5. 未成年人数据特化（呼应教育行业 K12）

K12（小学/初中/高中）数据天然涉及**未成年人**。我国《未成年人保护法》《个人信息保护法》对未成年人信息有特殊保护要求，这是**法律红线**，比一般 PII 更严格：

- 未成年人信息一律视为**高敏数据**，即使只是姓名+年级。
- 分析 K12 数据时，禁止在结论中呈现可识别到具体学生的信息（姓名+学校+年级组合可能定位到人）。
- 聚合统计优先（如"三年级共 N 人"），明细数据谨慎。
- 详见 `industry/education/k12.md`。

## 6. 反模式（禁止）

- ❌ **`SELECT *` 含 PII 的表**——把不需要的手机号、身份证一起拉出来，增加泄露面。
- ❌ **把含手机号/身份证的结果集明文贴进对话**——对话记录可能被留存、被看到。
- ❌ **明文导出含 PII 的文件**——文件一旦流出不可控。
- ❌ **把样例数据里的真实姓名/手机号原样展示**——样例也是真实数据。
- ❌ **未成年人数据当普通数据处理**——违法风险。
