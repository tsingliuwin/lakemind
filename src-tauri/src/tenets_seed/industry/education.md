# 教育行业数据分析准则

教育行业的核心数据围绕**课程销售**展开：体验课、正价课、VIP 服务并存，单价差异极大，转化链路复杂。以下是教育场景高频踩中的分析陷阱。

## 案例：体验课订单归因错误（致命）

这是真实发生过的错误，是本准则库的缘起案例，务必牢记。

**背景**：用户问"体验课有多少单"。订单表里有 `amount`（金额）和班型字段。

**错误的推理链**：
1. agent 先查到"全表中 `amount = 99` 的订单共 **3,546 单**"（这条查询的筛选条件是 `amount = 99`，覆盖所有班型）。
2. 然后直接断言"**体验课有 3,546 单 99 元**"——把"所有 99 元订单"的数字挪用到了"体验课"头上。

**真相**：
- 体验课绝大部分是 **0 元**，少量 0.1 元活动价和 29.9 元特价课。
- `amount = 99` 的订单里，**体验课只有 20 单**（是 1 对 1 辅导体验服务）。
- 其余 **3,526 单** 99 元是择校 VIP 等其他班型，与体验课无关。

**错在哪**：筛选条件（`amount = 99`，覆盖全表）和对象（"体验课"，需再按班型筛选）不一致。数字 3,546 属于"全表 99 元订单"，不属于"体验课"。这正是核心准则第 2 条的张冠李戴，详见 `/core/data-discipline.md`。

**正确做法**：要回答"体验课有多少单"，必须用同时按班型筛选的查询，而不是复用按金额筛选的结果。

## 教育行业通用陷阱

### 班型是核心维度，不能丢
体验课 / 正价课 / VIP 服务是教育数据的核心分类维度。任何"订单数""金额"类指标，**必须先按班型拆分**，否则会把单价差异极大的不同业务混在一起，得出无意义的合计。

### 体验课单价跨度极大
体验课可能同时存在 0 元、0.1 元、29.9 元、99 元等多种单价。说"体验课均价"前，先确认分位分布，警惕极端值拉偏。

### "免费"≠"无价值"
0 元体验课是获客漏斗的入口，它的价值在于后续转化，不能因为金额为 0 就在销售分析里忽略它。
