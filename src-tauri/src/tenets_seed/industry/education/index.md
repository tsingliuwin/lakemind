# 教育行业数据分析准则（总则）

教育行业的核心数据围绕**课程销售**展开：体验课、正价课、VIP 服务并存，单价差异极大，转化链路复杂。本文件是教育行业的总纲，各子行业（K12、考研等）差异巨大，具体准则见子目录文件。

## 教育行业核心维度

以下维度在教育行业的数据分析中必须优先关注，缺少任何一个都可能导致分析结论错误：

### 班型 / 课程类型
体验课、正价课、VIP 一对一、集训营、刷题班等是教育数据的**核心分类维度**。任何"订单数""金额"类指标，**必须先按班型拆分**，否则会把单价差异极大的不同业务混在一起，得出无意义的合计。

### 付费模式
教育行业的付费模式多样：免费体验、低价体验（0.1 元、9.9 元）、正价购买、分期付款、团购、奖学金减免等。分析销售数据时必须明确"金额"字段是原价、实付、还是含退款。

### 学段 / 学科
K12（小学/初中/高中）、大学、考研、职业教育等学段差异极大。同一机构可能跨多个学段运营，混合统计毫无意义。

### 转化链路
教育行业典型转化链路：广告曝光 → 落地页访问 → 领取体验课 → 完成体验 → 购买正价课 → 续费/升级。每个环节的转化率都有行业特性（详见 `topic/conversion.md`）。

## 教育行业通用陷阱

### 班型是核心维度，不能丢
体验课 / 正价课 / VIP 服务是教育数据的核心分类维度。任何"订单数""金额"类指标，**必须先按班型拆分**，否则会把单价差异极大的不同业务混在一起，得出无意义的合计。

### 体验课单价跨度极大
体验课可能同时存在 0 元、0.1 元、29.9 元、99 元等多种单价。说"体验课均价"前，先确认分位分布，警惕极端值拉偏。

### "免费"≠"无价值"
0 元体验课是获客漏斗的入口，它的价值在于后续转化，不能因为金额为 0 就在销售分析里忽略它。

## 缘起案例：体验课订单归因错误（致命）

这是真实发生过的错误，是本准则库的缘起案例，务必牢记。详见 K12 子行业准则（`industry/education/k12.md`）。

**简述**：agent 把"全表中 amount = 99 的订单共 3,546 单"直接说成"体验课有 3,546 单 99 元"——筛选条件和归属对象不一致，数字张冠李戴。实际上体验课 99 元的只有 20 单。
